# 🎯 AI 코딩 자동화 로드맵 생성 프롬프트

> **사용 대상 모델**: Claude Opus 4.6 / ChatGPT 5.3 (고급 추론 모델)
> **생성일**: 2025-02-15
> **적용 프레임워크**: 프롬프트 엔지니어링 8기법 조합

---

## 📋 프롬프트 (복사해서 사용)

```
당신은 비전공 초보자를 위한 AI 코딩 도구 전문 멘토입니다.
다음 3가지 도구를 활용한 자동화 프로그램 개발 학습 로드맵을 설계해주세요:

1. **OpenAI Codex** — 클라우드 기반 코딩 에이전트 (ChatGPT 연동, Codex CLI, IDE 확장)
2. **Claude Code** — Anthropic의 터미널 기반 에이전틱 코딩 도구 (MCP 연동)
3. **Google Antigravity** — 에이전트 중심 IDE (Gemini 3 Pro 기반, 브라우저 자동화 포함)

---

### 학습자 프로필
- 20세, 예고 졸업생 (비전공, 코딩 경험 거의 없음)
- 현재 사용 중인 도구: Obsidian, Notion, n8n, Claude, ChatGPT, Gemini, Perplexity
- 목표: AI 코딩 도구를 활용하여 반복 업무 자동화 및 웹 서비스 프로토타입 제작
- 학습 가능 시간: 하루 2~3시간
- 운영체제: macOS

---

### 요청사항

**[Phase 구조]** 3개월(12주)을 4개 Phase로 나누어 설계하세요:

- **Phase 1 (1~3주)**: 기초 환경 구축 및 핵심 개념
  - 각 도구의 설치/설정 방법
  - 터미널, Git, Node.js, Python 등 필수 기초
  - 첫 번째 "Hello World" 수준의 실습

- **Phase 2 (4~6주)**: 도구별 핵심 기능 실습
  - Codex로 코드 생성/리뷰/디버깅 실습
  - Claude Code로 MCP 연동 및 파일 작업
  - Antigravity로 에이전트 기반 웹앱 프로토타입

- **Phase 3 (7~9주)**: 자동화 프로젝트 실전
  - 3가지 도구를 조합한 실전 자동화 프로젝트 3개 제안
  - 각 프로젝트는 난이도를 점진적으로 높일 것
  - n8n 워크플로우 연동까지 포함

- **Phase 4 (10~12주)**: 통합 및 고도화
  - 3가지 도구의 강점별 사용 전략 정리
  - 개인 자동화 시스템 구축 (Obsidian + n8n + AI 코딩 도구)
  - 포트폴리오용 최종 프로젝트

---

### 각 Phase 출력 형식

Phase별로 아래 구조를 반드시 따르세요:

**[Phase N: 제목]**
- 🎯 학습 목표: 이 단계에서 달성할 수 있는 것
- 📚 선행 지식: 이 단계를 시작하기 위해 알아야 할 것
- 🛠️ 주차별 학습 내용:
  - N주차: 구체적 학습 항목 + 실습 과제
- 💻 실습 프로젝트: 해당 Phase의 미니 프로젝트 (구체적 설명 포함)
- ✅ 체크리스트: 다음 Phase로 넘어가기 전 자가 점검 항목
- ⚠️ 초보자 주의사항: 자주 막히는 지점과 해결법
- 📎 추천 리소스: 참고할 공식 문서, 영상, 가이드 (한국어 우선)

---

### 제약 조건

1. 모든 설명은 비전공 초보자가 이해할 수 있는 수준으로 작성하세요.
2. 전문 용어가 나오면 반드시 괄호 안에 쉬운 설명을 추가하세요.
3. 각 도구의 2025년 최신 기능과 변경사항을 반영하세요.
4. "바이브 코딩(Vibe Coding)" 방식의 접근을 기본으로 하되, 기초 개념 이해도 병행하세요.
5. 각 도구의 무료/유료 범위를 명시하세요.
6. 한국어로 작성하세요.

---

### 사고 과정

로드맵을 작성하기 전에 다음을 먼저 분석하세요:
1. 세 도구(Codex, Claude Code, Antigravity)의 핵심 차별점과 시너지 포인트
2. 초보자가 가장 먼저 배워야 할 공통 기초는 무엇인가
3. 어떤 순서로 도구를 도입해야 학습 곡선이 완만한가
4. 각 Phase의 난이도가 자연스럽게 상승하는지 검증

이 분석을 바탕으로 로드맵을 설계하세요.
```

## 📌 사용된 기법 및 근거

| # | 기법 | 선택 이유 |
|---|---|---|
| 1 | **역할 부여 (Role Play)** | "비전공 초보자를 위한 AI 코딩 도구 전문 멘토" — 전문성 + 눈높이 조절 동시 확보 |
| 2 | **Skeleton-of-Thought** | 4 Phase 뼈대를 먼저 지정하여 전체 구조의 논리적 일관성 유지 |
| 3 | **Modular Chaining** | Phase별 독립 모듈로 쪼개어 단계적 완성도 확보 |
| 4 | **Chain-of-Thought** | "사고 과정" 섹션에서 분석 → 설계 순서를 강제하여 논리적 추론 유도 |
| 5 | **Contextual Embedding** | 학습자 프로필(나이, 배경, 도구, OS)을 명시하여 맞춤형 답변 유도 |
| 6 | **템플릿/포맷 지정** | 각 Phase의 출력 형식을 7개 항목으로 고정하여 일관된 구조 보장 |
| 7 | **Negative Constraints** | 제약 조건에서 "전문 용어 그대로 사용 금지", "한국어 작성" 등 배제 사항 명시 |
| 8 | **Step-Back Prompting** | 로드맵 작성 전 "세 도구의 차별점과 시너지" 분석을 선행하도록 유도 |

## 💡 활용 팁

### Claude Opus 4.6에서 사용할 때
- 그대로 복사 붙여넣기하면 됩니다
- 응답이 길어지면 "Phase 1~2만 먼저 작성해줘" → "이어서 Phase 3~4 작성해줘"로 나눠서 요청

### ChatGPT 5.3에서 사용할 때
- 프롬프트 그대로 사용 가능
- Codex와 연동하여 실습할 때는 ChatGPT 사이드바의 Codex 기능을 바로 활용 가능
- "web search를 활용해 각 도구의 최신 정보를 반영해줘"를 추가하면 더 정확

### 후속 프롬프트 (Iterative Refinement)
프롬프트 결과를 받은 후 추가로 사용할 수 있는 후속 질문:

1. "Phase [N]의 실습 프로젝트를 더 구체적으로 설명해줘. 파일 구조, 명령어, 예상 결과물 포함."
2. "내가 [특정 단계]에서 막혔어. 이 에러 메시지를 해결하는 방법을 알려줘: [에러 내용]"
3. "이 로드맵에서 n8n 워크플로우와 연동하는 부분을 더 자세히 확장해줘."
4. "각 도구의 비용 비교표를 만들어줘. 무료 범위와 유료 전환 시점 포함."
5. "Phase 4의 최종 프로젝트를 Obsidian + n8n + Codex를 조합한 개인 지식관리 자동화 시스템으로 구체화해줘."
