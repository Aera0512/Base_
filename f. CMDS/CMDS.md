## Why This System Exists

CMDS (Connect ‚Üí Merge ‚Üí Develop ‚Üí Share) is a comprehensive Personal Knowledge Management (PKM) system designed to transform how you capture, process, and share knowledge. - **Learning System**: Framework for continuous growth and development

## How to Use This System

### For New Notes
1. Identify which CMDS category it belongs to (100-900)
2. Choose appropriate subcategory (N01-N99)
3. Use proper frontmatter with required properties
4. Link to related notes using wikilinks
5. Apply appropriate tags

### For Processing Information
1. **Capture** in `00. Inbox/` first
2. **Process** and categorize into appropriate CMDS folder
3. **Refine** and add connections
4. **Output** when ready to share

### For Creating Outputs
1. **Connect** - Gather related notes from 100 Themes
2. **Merge** - Review relevant literature from 200 Literature
3. **Develop** - Apply data and methods from 300-600
4. **Share** - Create deliverable in 700-800

## Best Practices

### 1. Start in Inbox
Always capture new information in `00. Inbox/` first. Process it later into the appropriate CMDS category.

### 2. Use Consistent Properties
Every note should have:
- `type:` - Note type
- `aliases:` - Alternative names
- `author:` - Author (usually `"[[Íµ¨ÏöîÌïú]]"`)
- `date created:` - Creation date (YYYY-MM-DD)
- `tags:` - Relevant tags

### 3. Link Liberally
Use wikilinks `[[]]` to connect related notes. The value of PKM increases with connections.

### 4. Review Regularly
- Daily: Process inbox
- Weekly: Review and refine notes
- Monthly: Identify gaps and opportunities

### 5. Evolve the System
The CMDS framework is flexible. Adapt it to your needs while maintaining the core structure.
